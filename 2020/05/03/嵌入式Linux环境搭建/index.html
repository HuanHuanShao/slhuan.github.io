1. 基础环境

  我使用的是VMWare Workstation 15 Pro和Ubuntu 20.04，关于VMWare和UI吧ubuntu的安装我就不多说了，本篇重点在于Ubuntu环境的设置以及交叉编译环境的搭建。

2. Ubuntu环境设置

2.1 修改软件源

2.1.1 备份初始源文件

    1.sudo cp /etc/apt/sources.list /etc/apt/sources.list.backup
### 2.1.2 编辑源文件 
    1.sudo gedit /etc/apt/sources.list
将源文件中的内容删掉，并替换为如下内容： 
# deb cdrom:[Ubuntu 16.04 LTS _Xenial Xerus_ - Release amd64 (20160420.1)]/ xenial main restricted
deb-src http://archive.ubuntu.com/ubuntu xenial main restricted #Added by software-properties
deb http://mirrors.aliyun.com/ubuntu/ xenial main restricted
deb-src http://mirrors.aliyun.com/ubuntu/ xenial main restricted multiverse universe #Added by software-properties
deb http://mirrors.aliyun.com/ubuntu/ xenial-updates main restricted
deb-src http://mirrors.aliyun.com/ubuntu/ xenial-updates main restricted multiverse universe #Added by software-properties
deb http://mirrors.aliyun.com/ubuntu/ xenial universe
deb http://mirrors.aliyun.com/ubuntu/ xenial-updates universe
deb http://mirrors.aliyun.com/ubuntu/ xenial multiverse
deb http://mirrors.aliyun.com/ubuntu/ xenial-updates multiverse
deb http://mirrors.aliyun.com/ubuntu/ xenial-backports main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ xenial-backports main restricted universe multiverse #Added by software-properties
deb http://archive.canonical.com/ubuntu xenial partner
deb-src http://archive.canonical.com/ubuntu xenial partner
deb http://mirrors.aliyun.com/ubuntu/ xenial-security main restricted
deb-src http://mirrors.aliyun.com/ubuntu/ xenial-security main restricted multiverse universe #Added by software-properties
deb http://mirrors.aliyun.com/ubuntu/ xenial-security universe
deb http://mirrors.aliyun.com/ubuntu/ xenial-security multiverse    
deb http://cz.archive.ubuntu.com/ubuntu trusty main universe
保存后，在终端更新： 
1.sudo apt-get update
对于想要学习使用Vim的这个时候可以去安装完整版的Vim了，不过在ubuntu 18后的版本，vim貌似都已经安装好了： 
1.sudo apt-get install vim
## 2.2 配置网络服务 ### 2.2.1 安装ftp、ssh、NFS 
sudo apt-get install vsftpd openssh-server nfs-kernel-server rpcbind
如果安装过程中遇到如下提示： 
  
    Reading package lists... Done
    Building dependency tree       
    Reading state information... Done
    Some packages could not be installed. This may mean that you have
    requested an impossible situation or if you are using the unstable
    distribution that some required packages have not yet been created
    or been moved out of Incoming.
    The following information may help to resolve the situation:

    The following packages have unmet dependencies:
     openssh-server : Depends: openssh-client (= 1:7.2p2-4ubuntu2.8)
                      Depends: openssh-sftp-server but it is not going to be installed
                      Recommends: ssh-import-id but it is not going to be installed
    E: Unable to correct problems, you have held broken packages.
这是因为openssh-server是基于openssh-client 1:7.2p2-4ubuntu2.8的，原装的client版本不支持，所以需要先安装一遍open-client覆盖原装的： 
sudo apt-get install openssh-client=1:7.2p2-4ubuntu2.8
然后再运行一遍刚开始的那一行命令就可以了。 ### 2.2.2 配置FTP 1. 打开vsftpd.conf: 
sudo gedit /etc/vsftpd.conf
2. 查看28行和31行首是否有#，有则去之： 
28.local_enable=YES ```  
31.#write_enable=YES --> write_enable=YES
### 2.2.3 配置NFS 1. 修改exports： 
sudo gedit /etc/exports
2. 增加如下内容： 
    /work                     *(rw,sync,no_root_squash,no_subtree_check)
    /work/slhuan/nfs_rootfs    *(rw,sync,no_root_squash,no_subtree_check)
我是在用户目录下新建了一个文件夹work，然后再在里面新建了文件夹slhuan： 
    slhuan@ubuntu:~$ mkdir work
    slhuan@ubuntu:~$ cd work/
    slhuan@ubuntu:~/work$ mkdir slhuan
目的是为了以后外部设备（比如开发板）可以通过网络文件系统访问work目录和专门用来作为根文件系统挂载的/work/slhuan/nfs_rootfs目录。 ![avatar][NFSstr] 3. 重启NFS服务： 
    sudo /etc/init.d/nfs-kernel-server restart
  关闭防火墙 
    sudo ufw disable

3. 安装基本开发环境

1.安装arm-linux
   此处使用的是arm-linux-gcc-4.4.3。这个压缩包放到你自己的文件夹下，然后将其解压到根目录：


    sudo tar xzf arm-linux-gcc-4.4.3.tar.gz –C /

2.修改环境变量 


    sudo gedit /etc/ environment

3.重启ubutu然后查看版本检验是否安装好： 


    sudo reboot
    arm-linux-gcc -v

如果安装成功，则会出现如下提示：

如果提示的是：

那么在一个目下新建一个脚本： 


gedit install-devel-packages-for-16.sh

内容如下： 



#!/bin/bash

####################################################################
# Warning:
# Ubuntu 16.04.1 introduced a new X stack that isn't compatible with
# with the old glx packages.

# ------------------------------------------------------------------
# 16.04                --> 16.04.1 (*-lts-xenial)
# ------------------------------------------------------------------
# libgl1-mesa-glx:i386 --> libgl1-mesa-glx-lts-xenial:i386
# libgl1-mesa-dev      --> libgl1-mesa-dev-lts-xenial
# ------------------------------------------------------------------
#
# DON'T install libgl1-mesa-glx:i386 on 12.04.2, else it will *BREAK*
# your system.
####################################################################

# Execute an action
FA_DoExec() {
    echo "==> Executing: '${@}'"
    eval $@ || exit $?
}

# Ubuntu 12.04.2 LTS
FA_DoExec apt-get install \
    git gnupg flex bison gperf build-essential \
    zip curl libc6-dev libncurses5-dev libncurses5-dev:i386 x11proto-core-dev \
    libx11-dev:i386 libreadline6-dev:i386 \
    libgl1-mesa-glx-lts-xenial:i386 libgl1-mesa-dev-lts-xenial \
    g++-multilib mingw32 tofrodos libncurses5-dev:i386 \
    python-markdown libxml2-utils xsltproc zlib1g-dev:i386


if [ ! -h /usr/lib/i386-linux-gnu/libGL.so ]; then
    FA_DoExec ln -s /usr/lib/i386-linux-gnu/mesa/libGL.so.1 \
        /usr/lib/i386-linux-gnu/libGL.so
fi

# Development support
FA_DoExec apt-get install \
    vim dos2unix minicom gawk


然后给这个脚本设置为可执行文件：  



chmod +x install-devel-packages-for-16.sh

执行这个脚本：  



sudo ./install-devel-packages-for-16.sh

最后再执行  



arm-linux-gcc -v

就会提示版本号，然后交叉编译环境安装完毕。
